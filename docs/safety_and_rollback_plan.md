This document details the safety gates and rollback plan for our migration system. The system enforces strict rules to prevent any accidental loss of files. The first safety gate is that no files or folders are moved unless they have a status of OK in the migration report. This means that only folders that have been fully copied and match the source size exactly will be touched. The second gate is that the move script runs in dry run mode by default to prevent any accidental execution. A rollback capability is built into the script to ensure you can revert the staging operation if anything goes wrong. The script logs the original path and the new path of every file and folder it moves to a transaction log in JSON format. If a rollback is triggered, the script reads this log and moves all files and folders back to their exact original locations. This rollback also runs in dry run mode by default, so you can verify the reversal before it executes.
