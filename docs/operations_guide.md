This operations guide describes how to run the scripts in this project. To begin a migration cycle, you must first run the manifest scanner script under the src folder. This script traverses the source and target folders to map all files and identify audiobook folders. You run it with python src/manifest_scanner.py passing the path to the drive. Once the scan is complete and the manifests are saved to the workspace root, you run the verify migration script using python src/verify_migration.py to compare the source and target manifests. This script outputs a validation report showing which folders are successfully copied and which are missing or have mismatched sizes. Finally, you run the safe move script using python src/safe_move.py with the source, staging, and empty folder staging paths. By default, the move script runs in dry run mode to show you what moves it would make. You must pass the execute flag to perform the actual move. If you need to undo the moves, you run the safe move script with the rollback flag and point it to the generated json log file.
