This document serves as the high level confluence overview for the audiobook migration and verification system. The system was designed to handle the safe migration of audiobook files from a pCloud network mount to Google Drive. The architecture is split into three main components which are the heuristic scanner, the migration verifier, and the safe move cleanup utility. The scanner uses file extensions, folder path patterns, metadata, and duration filters to distinguish audiobooks from music or system sounds. The verifier normalizes paths and compares exact file sizes to ensure nothing was lost during the copy process. The safe move utility stages successfully migrated folders to a temporary delete folder and isolates empty directory structures for manual audit. External auditors and stakeholders can review the migration report and move logs to verify that all files are accounted for and no unauthorized deletions occurred.
